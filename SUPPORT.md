# Support

## How to get help

- Open a GitHub Discussion (recommended for usage questions and ideas).
- Open a GitHub Issue for reproducible bugs or feature requests.

## Before opening an issue

- Confirm you are using a recent action tag (for example `@v1`).
- Include your workflow snippet and `tracker.json` sample.
- Include logs/traceback from the failed run.

## Scope

This project provides support for:
- Action usage and configuration
- Bug reports
- Feature requests

It does not provide support for:
- Organization-specific GitHub permission policies
- Private infrastructure constraints outside this repository
